# Important Points – Programming & Computer Basics

## 1. Computer

* A **computer is a machine that follows instructions**.
* It performs tasks by executing a sequence of commands.
* It is **fast, accurate, and obedient**, but has **no thinking ability of its own**.
* It only does **exactly what it is instructed to do**.

---

# 2. Hardware

* **Hardware** refers to the **physical components of a computer**.
* These components can be **seen and touched**.

Examples:

* Monitor
* Keyboard
* Mouse
* CPU cabinet
* Internal wires and fans

Key Idea:

* Hardware = **Physical parts of a computer system**

---

# 3. Software

* **Software** refers to the **programs or instructions that run on a computer**.
* Software cannot be touched but **controls how hardware works**.

Examples:

* Operating systems (Windows, Android, iOS)
* Applications (WhatsApp, YouTube)
* Games
* Calculator

Key Idea:

* Software = **Instructions stored inside the computer**

---

# 4. Relationship Between Hardware and Software

* Hardware and software **work together to operate a computer**.
* Without software → hardware cannot perform tasks.
* Without hardware → software cannot run.

Analogy:

* **Hardware = Body**
* **Software = Mind**

---

# 5. CPU (Central Processing Unit)

* The **CPU is the brain of the computer**.
* It **processes instructions and performs calculations**.
* It controls the operations of the entire computer system.

Key Points:

* Executes programs
* Performs logical and arithmetic operations
* Determines the speed of processing

---

# 6. RAM (Random Access Memory)

* **RAM is temporary memory used while programs are running**.
* It stores data that the CPU needs immediately.

Characteristics:

* **Fast memory**
* **Volatile (data is lost when power is off)**
* Limited capacity

Analogy:

* **RAM = Study table**

---

# 7. Storage (Hard Disk / SSD)

* **Storage is permanent memory used to store data and programs**.
* Data remains even when the computer is turned off.

Examples:

* Hard Disk Drive (HDD)
* Solid State Drive (SSD)

Characteristics:

* Large capacity
* Slower than RAM
* Non-volatile

Analogy:

* **Storage = Bookshelf**

---

# 8. Working of CPU, RAM, and Storage

The process of running a program:

1. Program is stored in **Storage**
2. Program loads into **RAM**
3. **CPU executes the instructions**
4. Output appears on the screen

---

# 9. Program

* A **program is a set of instructions that tells the computer what to do**.
* Programs are written using **programming languages**.

Examples:

* Calculator app
* Alarm app
* Messaging apps
* Games

---

# 10. Programmer

* A **programmer is a person who writes programs**.
* Also called:

  * Developer
  * Coder
  * Software Engineer

---

# 11. Bug

* A **bug is an error or mistake in a program**.
* It causes incorrect results or unexpected behavior.

Examples:

* Application crash
* Game freezing
* Incorrect output

---

# 12. Debugging

* **Debugging is the process of finding and fixing bugs in a program**.
* Programmers spend significant time debugging their code.

---

# 13. Binary Language

* Computers understand only **binary language**.

Binary consists of:

```
0 and 1
```

Meaning:

* 0 → OFF
* 1 → ON

Example:

```
01001000 01100101 01101100 01101100 01101111
```

This is called **machine language**.

---

# 14. Programming Languages

* Programming languages are used to **write instructions in a human-readable format**.
* These instructions are later converted into **binary for the computer**.

Examples:

* Python
* Java
* C++
* JavaScript

Purpose:

* Make programming easier for humans.

---

# 15. Why Multiple Programming Languages Exist

Different languages are designed for different purposes.

Examples:

* **Python** → AI, Data Science
* **JavaScript** → Web Development
* **C/C++** → System programming, games
* **Swift** → iPhone apps

---

# 16. Operating System

* The **Operating System (OS)** is the main software that manages computer hardware and software.

Examples:

* Windows
* Android
* iOS
* Linux

Functions:

* Managing memory
* Managing processes
* Handling files
* Controlling devices

---

# 17. Real-Life Programs

Many everyday systems run programs:

Examples:

* ATM machines
* Messaging apps
* Food delivery apps
* Traffic signal systems

All these systems are **controlled by software programs written by programmers**.

---

# 18. Learning Path in Programming

Typical journey of a programmer:

1. Learn a **programming language**
2. Study **data structures**
3. Learn **algorithms**
4. Build **projects**
5. Work on **real-world applications**

---

| Term       | Meaning                           |
